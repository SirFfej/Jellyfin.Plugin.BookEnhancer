Metadata Sources

Enrich your book library with metadata from online sources. The plugin extracts ISBN from book files and looks up full metadata from the sources below, cascading from highest quality to fallback.

If an API is rate-limited, the plugin will wait up to this many seconds for a slot before falling back to the next source. Set to 0 to wait indefinitely (legacy behavior).

Hardcover

Primary source — best quality metadata including authors, narrators, translators, illustrators, series, genre tags, ratings, ASIN, and more. Requires a free API key.

Paste your Hardcover API key here.
How to get a Hardcover API Key
  1. Go to hardcover.app/account/api
  2. Sign in to your Hardcover account (or create one — it's free, no subscription required)
  3. Click Generate API Key
  4. Copy the generated key and paste it in the field above
  5. Rate limit: 60 requests per minute, free tier
  6. Note: API tokens expire annually on January 1st — generate a new one each year
A Hardcover API key is strongly recommended — it provides the best quality metadata (authors, narrators, series, genres). The plugin works without it by falling back to Google Books and OpenLibrary.

Google Books

Fallback source — good metadata coverage.

You do NOT need an API key — the plugin works without one (IP-based rate limiting applies). If you have a key: Get one at Google Cloud Console , enable the Books API, then paste it here for a higher rate limit (1,000 requests/day).

OpenLibrary

Final fallback — free, no API key required. Covers many public domain and older titles not found in other sources.

Comic Vine

Comic-specific metadata source — ideal for single-issue comics. Provides creative team (penciller, inker, colorist, etc.), characters, story arcs, cover art, and publisher info. Free API key required.

Paste your Comic Vine API key here. Get one at Comic Vine
Default 180/hour. Increase this if your Comic Vine account shows no current rate limits.

Metron

Community-run comic database alternative to Comic Vine. Provides creative team credits, characters, cover art, and publisher info. Uses HTTP Basic Auth with your Metron account credentials.

Enter your Metron account credentials (username/email and password). Sign up at Metron — create a free account, then enter your credentials here.

VerseDB

Modern comic database with excellent search and barcode lookup. Provides creative team credits, characters, cover art, and publisher info. Free account required for API token.

Paste your VerseDB API token here. Sign up at VerseDB — create an account, then go to your API settings to generate a token.

Grand Comics Database

Open comic database with comprehensive creative team credits, characters, cover art, and publisher info. CC-licensed data, 2M+ issues. Used as final fallback in the comic cascade. Free account required for higher rate limits (2000/day vs 30/hr anonymous).

Enter your comics.org account credentials. GCD uses HTTP Basic Auth for API access. Sign up at comics.org — free account raises rate limit from 30/hr to 2000/day.
Checks whether this server can reach Hardcover, Google Books, OpenLibrary, Comic Vine, Metron, VerseDB, and Grand Comics Database. Useful for diagnosing network issues.

ComicInfo Template

The plugin ships a default ComicInfo.xml template. Values you add to it are merged into CBZ comics that are missing those fields when metadata write-back runs. You can also place an XML file with the same base name as a CBZ next to it (for example Batman 01.xml beside Batman 01.cbz); that sidecar file is merged into the CBZ. CBR/CB7 files must be converted to CBZ first before the template or sidecar can be written into them.

On the next library scan/cleanup with metadata write-back enabled, those values are merged into CBZ comics that do not already have them. For per-book CBZ overrides, rename the file to match the comic file name and place it in the same folder.

Metadata Guide

Edit metadata in Jellyfin → next library scan → plugin writes to file tags → permanent.

✅ Edit Metadata now feeds back into the plugin.
The plugin reads from Jellyfin's DB after parsing file tags. If a file tag is missing, the plugin checks Jellyfin's DB — so any field you fill in Edit Metadata will be picked up on the next library scan. From there, the plugin writes it back into the file's native tags (ComicInfo.xml, EPUB OPF, ID3), making the edit permanent across DB resets.

Example: Fixing a file in "Unknown Publisher/Standalone"

Your file lands at Unknown Publisher / Standalone / Title / Title.cbz when Publisher and Series are missing. Here's how to fix it:

  1. Set Publisher (Studio): Edit Metadata → Main tab → Studios → type Image Comics — maps to {Publisher} token
  2. Set Series: Series is not shown in the Book Edit Metadata UI, so set it in the file's ComicInfo.xml (<Series>Scarlet</Series>) using ComicTagger or ComicRack before scanning
  3. Set Issue Number: Also not shown in the UI — set <Number>4</Number> in ComicInfo.xml before scanning
  4. Bulk fix with a template: If many files share the same missing Publisher, configure a ComicInfo template on the Metadata tab with <Publisher>Image Comics</Publisher> so every empty Publisher field inherits it without overwriting existing values.

After enrichment, the plugin writes enriched metadata back into the ComicInfo.xml. On next library scan, the template produces Image Comics / Scarlet / Scarlet 04.cbz.

Alternatively, let the comic API cascade (ComicVine → Metron → VerseDB → GCD) populate Series + Issue Number automatically — just ensure Studios is set in Edit Metadata for the {Publisher} directory.

Default template logic

If Publisher is set → {Publisher}/{Series}/{Title}.
Otherwise → {Author}/{Series}/{Title}.

Each managed directory can override this template in the Ingestion tab.

Multi-disc audiobooks

For audiobooks split into multiple discs, use {BookTitle} and {Disc} in the per-directory template. Examples:

  • {Author}/{Series}/{BookTitle}/{Disc} → Anne Perry/Standalone/Dark Assassin/Disc 08/Anne Perry - Dark Assassin Disc 08.mp3
  • {Author}/{Series}/{Disc} → Aldous Huxley/Brave New World/Disc 01/1a.m4a

{BookTitle} is derived from the audio Title tag by removing the author prefix and disc marker. {Disc} is extracted from markers like Disc 08, [Disc 1], CD 2, or Part 3 of 12.

Comic Library toggle

In the Ingestion tab, expand any managed directory and check Comic Library to treat every file in that source as a comic. This enables filename-based series/issue parsing for PDFs (not just CBZ/CBR/CB7) and makes those files eligible for the ComicIssue grouping strategy and the comic enrichment cascade.

ComicInfo.xml template fallback

On the Metadata tab you can configure a ComicInfo template file (local path or downloadable URL). When a comic archive has empty or missing fields, the template supplies defaults (for example, setting Publisher to Marvel for an entire publisher dump). Template values never overwrite fields that already exist in the archive's ComicInfo.xml.

Grouping strategies

The Grouping tab lets you choose how alternate formats of the same item are matched:

  • ISBN only — exact ISBN match (recommended for books/audiobooks)
  • Title + Author — exact title and author match
  • FileNamePrefix — groups by series name or title prefix
  • ComicIssue — groups by parsed SeriesName + issue number. Use this for comic libraries to keep sub-series separate (e.g., Star Wars vs Star Wars - Darth Vader) while still grouping CBZ + CBR of the same issue.

Task runtime limits

The Main tab has Task Runtime Limits for Ingestion Scan, Library Cleanup, and Metadata Enrichment. When a task exceeds its limit it is cancelled gracefully. Ingestion Scan resumes from its last checkpoint on the next run; cleanup and enrichment simply stop and continue from where they left off on the next scheduled run.

Per-directory API selection

Each managed source directory can override the global API toggles. Expand a directory row in the Ingestion tab and check the APIs you want to allow for that source. If no APIs are checked, the directory falls back to the global toggles on the Metadata tab. This is useful when one library should only use comic APIs and another should only use book APIs, or when you want to disable online enrichment entirely for a specific source.

Enrichment cooldown

The Skip re-enrichment if already enriched within (days) setting on the Main tab prevents the same file from being hammered on every scheduled task run. When a file is skipped because of cooldown, the log entry includes the API that last enriched it, for example: Skipped enrichment (cooldown, last by Hardcover).
